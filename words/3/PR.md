# 📌 쿠버네티스 개념 학습

## 📖 참고 링크: https://kubernetes.io/ko/docs/concepts/

## 1. 쿠버네티스란?

- 컨테이너화된 워크로드와 서비스를 관리하기 위한 이식 가능하고 확장 가능한 오픈소스 플랫폼.

- 컨테이너 오류 처리, 확장, 배포 등을 시스템적으로 지원.

- YAML 기반 설정을 통해 배포 및 관리를 수행.

## 2. 쿠버네티스 주요 기능

- 서비스 노출: DNS 이름 또는 자체 IP를 통해 컨테이너 접근 가능.

- 로드밸런싱: 트래픽 분산 제공.

- 스토리지 오케스트레이션: 다양한 저장소 시스템 사용 가능.

- 자동 롤아웃/롤백: 새 컨테이너 배포 및 이전 버전으로 되돌리기 가능.

- 자원 관리(빈패킹): CPU, RAM 등 리소스 최적 할당.

- 자가 복구: 컨테이너 장애 발생 시 자동 교체.

- 시크릿 & 구성 관리: 환경변수, 암호 등 보안 값 관리.

- 수평 확장: 명령어, UI, CPU 사용량에 따라 오토스케일링.

## 3. 쿠버네티스가 아닌 것

- 빌드 도구 아님 → 빌드된 이미지를 실행하고 관리만 함.

- CI/CD 워크플로우 포함하지 않음 → 조직 문화/도구에 따라 구성 필요.

- 로깅/모니터링 솔루션 내장 아님 → 외부 도구 연계 필요.

- 단순 오케스트레이션 시스템이라기보다 오케스트레이션 필요성을 줄이는 플랫폼.

## 4. 클러스터 아키텍처

- 클러스터 구성: 컨트롤 플레인(Control Plane) + 워커 노드(Node).

- 컨트롤 플레인: kubectl 명령어를 통해 API 서버로 요청 → 노드로 전달.

- 노드(Node): kubelet이 명령 수행 및 결과 반환.

- 서비스(Service): Pod 집합에 접근할 수 있는 게이트웨이 역할 + 로드밸런싱.

- Ingress: 서비스보다 상위 개념, 외부 트래픽을 내부 서비스로 라우팅하는 API Gateway.

- 스토리지:

  - PV(Persistent Volume): Pod 데이터 영구 저장 리소스.

  - PVC(Persistent Volume Claim): PV를 Pod에서 제어하는 방식.

- ConfigMap/Secret: 환경 변수, 설정 값, 보안 값 관리.
