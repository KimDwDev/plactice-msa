# 📌 쿠버네티스 개념 학습

## 📖 참고 링크: https://kubernetes.io/ko/docs/concepts/

## 1. 쿠버네티스 개념

- 쿠버네티스는 컨테이너화된 워크로드와 서비스를 관리하기 위해 이식할 수 있고, 확장 가능한 오픈소스 플랫폼을 말한다. 

- 쿠버네티스는 컨테이너 오류나 확장을 할때 시스템 적으로 처리할 수 있게 해준다.

- 쿠버네티스는 배포 패턴을 제공하며 쿠버네티스는 배포를 쉽게 관리할 수 있다.

- 쿠버네티스는 yaml을 사용하여 쿠버네티스를 빌드할 수 있다.

## 2. 쿠버네티스 제공 기능

- 쿠버네티스는 DNS이름을 사용하거나 자체 IP주소를 사용하여 컨테이너를 노출 할 수 있다.

- 쿠버네티스는 트래픽에 대한 로드밸런싱을 제공한다.

- 쿠버네티스는 스토리지 오케스트레이션이 가능해서 원하는 저장소 시스템을 사용할 수 있다.

- 쿠버네티스에는 자동화된 롤아웃으로 배포용 새 컨테이너를 만들고 롤백이 가능하도록 한다.

- 쿠버네티스는 빈패킹을 사용하여 각 컨테이너가 필요로 하는 CPU와 RAM을 지시하고이를 제공하게 할 수 있다.

- 쿠버네티스의 컨테이너가 만약 다운되면 자동화된 복구를 통해서 컨테이너를 교체할 수 있다.

- 쿠버네티스의 안에서 만약 환경변수나 암호 등과 같은 것을 시크릿과 구성관리를 할 수 있다.

- 쿠버네티스는 간단한 명령어, UI 또는 CPU 사용량에 따라서 수평으로 확장을 할 수 있다.

## 3. 쿠버네티스가 아닌 것

- 쿠버네티스는 자동으로 빌드해주는 도구가 아니다 -> 빌드한 것을 실행하도록 도와주는 것이다.

- CI/CD 워크플로우는 쿠버네티스에서 해결해주는 것이 아닌 사내 문화에 맞게 작성하여야 한다.

- 로깅, 모니터링과 같은 경보 솔루션을 포함하지 않는다. 수집하고 노출하는 메커니즘은 집적 작성해야 한다.

- 쿠버네티스는 오케스트레이션 시스템이 라고 부르기에는 많은 기능이 있지만 오케스트레이션이 필요없게 만드는 것이 바로 쿠버네티스라고 할 수 있다.

## 4. 클러스터 아키텍처

- 쿠버네티스 클러스터에는 컨트롤 플레인과 노드라고 부르는 워커 머신으로 구성되어 있다.

- 유저는 kubectl을 통해서 컨트롤 플레인에 요청할 수 있고 이를 바탕으로 http, grcp등을 이용한 api를 통해서 node에게 명령을 내린다.

- node에는 kubelete이 이를 책임지며 결과를 반환한다.

- node에 서비스는 다양한 pod에 접근할 수 있게해주는 gateway라고 보면된다 이를 이용해서 로드밸런싱도 가능하다.

- node에 ingress는 service 보다도 상위 개념인데 서비스로 갈 수 있게 해주는 api gateway와 같은 역할이라고 보면 된다.

- pv는 디스크로 pod에 데이터를 영구적으로 저장하기 위한 리소스 이고 이를 pvc를 이용해서 제어해준다.

- configMap을 활용하여 환경변수등을 주입할 수 있다.
