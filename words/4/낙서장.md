# kafka 실습

- kafka에서 제공하는 브로커에 topic등을 설정하고 이를 통해서 브로커에 저장한다.

- kafka를 도커를 이용해서 컨테이너에서 작동하게 하는 방법은 좋은 방법이라고 생각한다.

- dockerFile로 kafka를 운영하고자 하면 환경변수가 너무 많아서 많이 번거롭다 그래서 docker-compose가 훨씬 효율적인 대안이 될 수 있다.

- kafka는 각 브로커에 따라서 개별적으로 브로커를 설정할 수 있어서 이를 이용해서 브로커를 이용할 수 있다.

- kafka는 docker-compose로 설정할 때 가장 신경써야 할 것 중하나가 바로 환경변수이다.

  - ALLOW_PLAINTEXT_LISTENER: kafka의 ssl인증 없이 바로 평문으로 데이터를 교환하고 싶을때 yes를 한다 보통은 no를 해서 ssl연결을 하도록 해야 보안적으로 안전하다

  - KAFKA_LISTENERS는 만약 브로커나 컨트롤러에 접근하고 싶다면 어떻게 접근해야 하는지 정하고 싶을때 사용한다.

  - KAFKA_ADVERTISED_LISTENERS 이건 외부에서 어떤 방법으로 접근할 것인지 정할때 사용해야 한다.

  - KAFKA_CONTROLLER_LISTENER_NAMES이를 이용해서 controller의 이름을 정할 수 있다.

  - 컨트롤러는 두뇌의 역할을 하면서 내부에서 진두지휘하는 역할을 맡는다.

  - KAFKA_LISTENER_SECURITY_PROTOCOL_MAP을 이용해서 어떻게 연결할 것인지 정할 수 있다.

  - kafka는 이제 더이상 예전처럼 zookeper를 이용해서 메타데이터를 관리하고 하지 않는다.

  - 브로커를 여러개를 둠으로써 안정성을 더욱 높일 수 있다.

## docker에 모든 데이터 삭제: docker system prune -a --volumes -f

```
모든 데이터 삭제
docker stop $(docker ps -aq)
docker rm $(docker ps -aq)

모든 이미지 삭제
docker rmi -f $(docker images -q)

네트워크 볼륨 삭제
docker network prune -f
docker volume prune -f
```

  - kafka에서 외부에 노출된건 브로커이다 컨트롤러는 내부에서 실행되고 있음으로 외부에서 알 필요는 없다.

  - docker exec -it <컨테이너네임> <명령어>는 안에 들어가서 명령을 실시 할 수 있다.

  - kafka는 mysql처럼 안에다가 topic도 집적 만들고 관리도 집적해주어야 한다.

  - ./kafka-topics.sh --bootstrap-server localhost:9092 --list 현재 토픽 리스트를 확인할 수 있게 해준다.

  - \는 bash에서 아직 줄이 바뀌지 않는다 이어진다는 것을 알려주는 것이다.

  - kafka에서 topic과 관련된 명령어들은 ./kafka-topics.sh에 존재한다. 매개변수를 줌으로써 topic과 관련된 명령어를 실행할 수 있다.

  - 아래와 같은 방법으로 topic을 생성할 수 있다.

```bash
/opt/kafka/bin/kafka-topics.sh --bootstrap-server localhost:9092 \  
--create \
--topic "토픽이름" \
--partitions "갯수" \
# 몇개의 브로커에 복제할지 정하는 매개변수
--replication-factor "갯수" \
```

  - 보통 한개의 브로커만 사용하는 경우는 개발용으로 사용하기에 적합하다 배포용으로는 더욱 견고한 앱을 위해서 잘 설계를 해주어야 하는 부분이다. 

  - kafka-console-consumer.sh나 kafka-console-producer.sh를 이용해서 cli로 consumer와 producer를 다룰 수 있다.

  - kafka는 메시지 기반으로 이를 기반으로 producer에서 이벤트를 만들면 consumer에서 처리가 가능하다.

  - kafka는 offeset을 기반으로 메시지를 소비할 수 있다. 

  - btoker 기반이라서 이전에 메시지도 모두 읽을 수 있다는 장점이 있다. 

  - 한글도 모두 가능하다는 점을 확인할 수 있었다.


- kafka를 nodejs에서 실행하기 위해서 kafkajs를 설치해야 한다.

- kafka를 nodejs에서 사용하기 위해서 접속하기 위한 port번호와 clientId가 필요하다. 

- new Kafka({clientId, brokers})를 이용해서 Kafka를 이용 가능하다.

- kafka의 새로운 토픽을 만들때는 공백이 들어가서는 안된다.
